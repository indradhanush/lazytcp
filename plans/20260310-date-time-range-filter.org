#+TITLE: lazytcp Date Time Range Filter

* Objective
Add a =Date Time= range filter category in =Filter By= so users can constrain packets by timestamp interval.

* Requested Behavior
- =Filter By= includes a date-time category.
- Popup provides editable =Start= and =End= time fields.
- Start and end are both optional.
- If only start is set, filter applies =timestamp >= start=.
- If only end is set, filter applies =timestamp <= end=.
- If both are set, filter applies inclusive range between bounds.
- Filter expression pane reflects the date-time range expression.

* Scope
In scope:
- Extend =FilterDimension= with date-time category.
- Add popup input mode for optional start/end values.
- Add range matching semantics for optional bounds.
- Add unit tests for popup candidates and range filtering.

Out of scope:
- Free-form timestamp input.
- Timezone conversion UX.

* Architecture Slice
- =src/domain.rs=
  - Add new dimension + labels.
- =src/app.rs=
  - Add date-time popup input mode and range-match behavior.
  - Build range-oriented expression string.
  - Add tests.
