#+TITLE: lazytcp Main-Branch Coverage Badge Update Plan

* Objective
Ensure =main= branch CI updates repository coverage badge data using this repo's =make coverage= flow, and expose a single machine-readable total coverage percentage.

* Scope
In scope:
- Extend =scripts/coverage.sh= to output a single total coverage percentage for the whole repo.
- Update =.github/workflows/ci.yml= to publish coverage badge data on =push= to =main=.
- Keep current Rust checks behavior for PR and push.

Out of scope:
- Enforcing coverage thresholds.
- Changing Rust test content.
- TUI-driven integration test automation.

* Implementation Steps
1. Add a deterministic coverage total artifact (line coverage percentage) to =target/coverage/= from the coverage script.
2. Add CI steps gated to =refs/heads/main= that:
   - run =make coverage=,
   - read the total percentage artifact,
   - determine badge color,
   - write badge state to gist via dynamic badge action.
3. Keep workflow behavior unchanged for non-main refs except existing checks.

* Verification
- Validate script output artifact exists and contains numeric percentage after =make coverage=.
- Validate workflow YAML shape and conditional logic by inspection.
