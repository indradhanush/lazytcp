#+TITLE: lazytcp Filter Popup Slash Search

* Objective
Enable in-popup substring search for value pickers so pressing =/= and typing narrows candidates without restarting the popup.

* Requested Behavior
- Open a value popup from the filter pane.
- Press =/= to enter search mode.
- Type text (for example =10.10=) to narrow visible options.
- Matching is substring-based (contains), not prefix-only.
- Existing selection toggles and apply flow continue to work.

* Scope
In scope:
- Multi-select popup state updates in =src/app.rs=.
- Popup key routing updates in =src/main.rs=.
- Popup rendering hints in =src/ui.rs=.
- App unit tests for search filtering semantics.

Out of scope:
- Date-time popup editing behavior.
- Persisted search history.

* Implementation Plan
1. Extend multi-select popup state with search query and base candidate list.
2. Add app methods to start/edit search and recompute visible candidates by case-insensitive substring.
3. Route =/= and typed characters in non-date popup mode.
4. Update popup footer/title to indicate search affordance and active query.
5. Add regression tests for contains-match narrowing and selection persistence under filtered views.
6. Run =cargo fmt=, =cargo test=, and =cargo check=.
